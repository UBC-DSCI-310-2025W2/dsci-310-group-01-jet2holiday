# Contributing

We welcome all contributions to this project!

## Reporting Issues
If you notice a bug or have a feature request, please open an issue [here](https://github.com/UBC-DSCI-310-2025W2/dsci-310-group-01-jet2holiday/issues).

To help us address your request efficiently, please ensure your report includes the following:

1. Descriptive Title
Use a concise title that summarizes the problem (e.g., `Error: ggplot2 fails to render in Docker`).

2. Bug Reports
If you are reporting a bug, please include:
- **Steps to Reproduce:** A clear list of steps to trigger the issue.
- **Expected vs. Actual Behavior:** What you thought would happen vs. what actually happened.
- **Error Messages:** Copy and paste any logs or terminal output.
- **Environment:** Are you running this locally or inside the Docker container?

3. Feature Requests
If you are requesting a new feature, please describe:
- The **motivation** for the change (e.g., "This will make the p-value calculation more robust").
- A brief description of the **proposed solution**.

## Core Members
1. Create a new branch from `main` for your changes:
```
   git checkout -b your-branch-name
```
2. Make your changes and commit them.
3. Push your branch and open a pull request against `main`.
4. Ensure your pull request includes a clear description of the bug fix or feature, and is assigned to the appropriate milestone (e.g. Milestone 1).
5. At least one other teammate must review and approve before merging.
6. Pull requests will be reviewed within 2 days.

## External Contributors
1. Open an issue first to describe what you would like to contribute.
2. Fork the repository and create a branch for your changes.
3. Submit a pull request against the `main` branch of the original repository.
4. Pull requests will be reviewed within 7 days.

## Code of Conduct
All contributors must abide by our [code of conduct](CODE_OF_CONDUCT.md).
